# 機能一覧：jyogiverse（サークル作品ホスティングサービス）

---

## 0️⃣ 前提定義

| 項目     | 内容                                              |
| ------ | ----------------------------------------------- |
| 対象ユーザー | サークルメンバー（申請者）/ 管理者（柳井）/ 作品閲覧者（一般）              |
| 提供価値   | バックエンド常時稼働が必要なWebアプリを、月額0円で公開・運用できる             |
| コア体験   | 申請フォームで申請 → 管理者が承認・デプロイ → `{作品名}.jyogiverse.dev` で即日公開 |
| 成功指標   | 同時10作品以上を常時稼働、メンバー1人あたり月額0円（電気代・ドメイン代除く）       |

---

# 1️⃣ 機能カテゴリ設計

---

## 【カテゴリA：コンテナ管理（Proxmox / LXC）】

| #   | 機能名               | 概要                                         | ユーザー価値                 | 優先度   | 備考           |
| --- | ----------------- | ------------------------------------------ | ---------------------- | ----- | ------------ |
| A-1 | LXCテンプレート作成       | 言語別（Node.js / Python / Go）のベーステンプレートを整備    | デプロイ手順を標準化できる          | 🔴 P0 | Debian 12ベース |
| A-2 | コンテナクローン          | `pct clone` でテンプレートから新規コンテナを1コマンドで複製       | 環境構築が即座に完了する           | 🔴 P0 |              |
| A-3 | systemdサービス登録・起動  | アプリをsystemdサービスとして起動・自動再起動管理する            | 障害時の自動復旧が保証される         | 🔴 P0 |              |
| A-4 | コンテナ一覧・状態確認       | 管理UIからコンテナ一覧と稼働状況を確認する                    | 管理者が状況を即座に把握できる        | 🟡 P1 | Phase 3以降    |
| A-5 | コンテナ起動・停止・再起動     | 管理UIのボタンからコンテナをコントロールする                   | 手動SSHなしで操作できる          | 🟡 P1 | Phase 3以降    |
| A-6 | リソース使用量確認         | CPU・メモリ・ディスク使用量を管理UIから確認する                | リソース不足を早期検知できる         | 🟢 P2 |              |
| A-7 | Git push 自動デプロイ   | push時にコンテナ内のコードを自動更新する                    | 手動デプロイ不要になる            | ⚪ P3  | Phase 4以降    |

---

## 【カテゴリB：外部公開（Caddy / Cloudflare Tunnel）】

| #   | 機能名                 | 概要                                       | ユーザー価値                   | 優先度   | 備考          |
| --- | ------------------- | ---------------------------------------- | ------------------------ | ----- | ----------- |
| B-1 | Cloudflare Tunnel設定 | ポート開放なしでサーバーを外部公開する                      | 自宅IPを公開せず安全に外部公開できる      | 🔴 P0 |             |
| B-2 | サブドメイン自動割当          | `{作品名}.jyogiverse.dev` 形式でURLを発行         | 作品ごとに独立したURLを即日取得できる     | 🔴 P0 |             |
| B-3 | Caddyリバースプロキシ設定     | 管理LXCのCaddyで各アプリLXCへルーティングする            | 追加コストなしで複数アプリを公開できる      | 🔴 P0 |             |
| B-4 | HTTPS自動化            | CaddyがLet's Encryptで自動TLS証明書取得・更新        | セキュアな通信が自動で確保される         | 🟡 P1 |             |
| B-5 | 独自ドメイン対応            | 独自ドメインをコンテナに割り当てる                        | 就活ポートフォリオに使いやすい          | ⚪ P3  | 初期スコープ外     |

---

## 【カテゴリC：申請・承認フロー（管理UI）】

| #   | 機能名             | 概要                                       | ユーザー価値                  | 優先度   | 備考      |
| --- | --------------- | ---------------------------------------- | ----------------------- | ----- | ------- |
| C-1 | 作品申請フォーム        | メンバーが言語・アプリ名・GitHubリポジトリURLを入力して申請する    | 手軽に公開申請できる              | 🔴 P0 | Phase 3 |
| C-2 | 申請一覧（管理者）       | 管理者が未承認申請を一覧で確認する                        | 申請見逃しを防げる               | 🔴 P0 | Phase 3 |
| C-3 | 承認・デプロイ操作       | 管理者がボタン1つでコンテナ作成→アプリ起動→URL発行を実行する       | デプロイ作業を大幅に簡素化できる        | 🔴 P0 | Phase 3 |
| C-4 | 申請ステータス確認（メンバー） | メンバーが自分の申請の進捗を確認できる                      | 進捗が可視化されて安心できる          | 🟡 P1 |         |
| C-5 | 却下・理由通知         | 管理者が申請を却下する際に理由を記載してメンバーに伝える            | 改善点がわかりリトライしやすい         | 🟡 P1 |         |
| C-6 | Slack通知         | 申請・承認・却下時にSlackへ自動通知する                   | 確認漏れを防げる                | 🟢 P2 |         |

---

## 【カテゴリD：作品公開ページ】

| #   | 機能名       | 概要                              | ユーザー価値             | 優先度   | 備考 |
| --- | --------- | ------------------------------- | ------------------ | ----- | -- |
| D-1 | 作品一覧ページ   | 公開中の作品をカード形式で一覧表示する             | サークル作品を発見しやすくなる    | 🟡 P1 |    |
| D-2 | 作品詳細ページ   | アプリの説明・作者・使用技術・GitHubリンクを表示する  | 作品の背景を閲覧者に伝えられる    | 🟡 P1 |    |
| D-3 | 稼働状況バッジ   | 各アプリのオンライン/オフラインステータスをリアルタイム表示  | 閲覧前に動作確認できる        | 🟢 P2 |    |

# 2️⃣ 優先順位の定義基準（共通ルール）

| 優先度   | 判断基準              |
| ----- | ----------------- |
| 🔴 P0 | これがないとプロダクトが成立しない |
| 🟡 P1 | 体験を大きく向上させる       |
| 🟢 P2 | 運用効率・継続率を改善する     |
| ⚪ P3  | 将来の拡張性・差別化        |

# 3️⃣ フェーズ整理

```text
【Phase 1 - インフラ基盤（MVP）】
🔴 A-1, A-2, A-3, B-1, B-2, B-3
→ 手動デプロイで1作品をURLで公開できる状態を目指す

【Phase 2 - デプロイスクリプト整備】

→ スクリプト1本で誰でも（管理者が）デプロイできる状態にする

【Phase 3 - 管理Web UI実装】
🔴 C-1, C-2, C-3
🟡 A-4, A-5, C-4, C-5, D-1, D-2, B-4

【Phase 4 - 拡張】
⚪ A-7, B-5
🟢 A-6, C-6, D-3
```
